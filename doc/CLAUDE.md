# 🧩 一、在现有结构上新增数据模型（重点）

## 1. SubTask（子任务）

子任务按“监测要素类型”划分：

类型固定为：
- 水质（water）
- 沉积物（sediment）
- 生物（biology）
- 渔业资源（fishery）

字段：

- id
- taskId
- type（子任务类型）
- indicators（该子任务的指标列表）
- status（pending / running / completed）

---

## 2. SubTaskExecution（子任务执行）

用于表示“某子任务在某站位的执行情况”

字段：

- id
- taskId
- subTaskId
- stationCode
- progress（0-100）
- status（未开始 / 进行中 / 已完成）

---

⚠️ 注意：
- 子任务 ≠ 站位
- 子任务是“任务拆解”
- 执行才和站位关联

---

## 2. 总任务进度

或：

= 所有 execution 完成比例

---
