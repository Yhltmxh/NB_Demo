根据任务书，实现一个系统，支持：
1. 任务结构化拆解

3. 严格的指标完整性校验
4. 进度统计与缺失分析

【技术要求】

- Vue 3 + Composition API + <script setup>

# 🧩 一、数据模型设计（重点）

请严格按真实业务建模：

## 1. Task（任务）
- id
- taskCode
- name
- startTime
- endTime

---

## 2. SubTask（子任务类型）
来自任务书：
- 疏浚泥监测
- 水质监测
- 沉积物监测
- 生物监测
- 渔业资源调查
- 环境监视

字段：
- id
- taskId
- type
- frequency（施工中/施工后等）

---

## 3. Station（站位）

示例：
- HD1 ~ HD15（疏浚泥）
- S11 ~ S20（水质等）

字段：
- id
- subTaskId
- stationCode
- hasLayers（是否分层）
- layers（如：表层、0.5m、1m、1.5m）
- monitoringElements（如：水质、沉积物、生物）

---

## 4. Indicator（指标定义）

必须结构化：

示例：
- 水质指标（pH、DO、COD等）
- 沉积物指标
- 生物指标

字段：
- id
- name
- category（如：水质/沉积物/生物）
- unit（可选）

---

## 5. DataRecord（数据录入）

核心数据结构：

- taskId
- subTaskId
- stationId
- layer（可选）
- indicatorId
- value
- status（已填写 / 缺失）

---

# ⚠️ 二、关键逻辑（必须实现）

## 1. 自动生成数据录入结构

根据：
任务 → 子任务 → 站位 → 层次 → 指标

👉 自动生成“应填写的数据矩阵”

---

## 2. 指标完整性校验（核心）

必须实现函数：

- validateTask(taskId)
- getMissingItems(taskId)
- getCompletionRate(taskId)

校验维度：

✔ 是否所有站位都有数据  
✔ 是否所有层次都有数据（柱状样）  
✔ 是否所有指标都填写  

---

## 3. 特殊规则（来自任务书）

必须体现：

### 疏浚泥：
- 柱状样站位（HD1、HD8、HD14） → 多层
- 其他 → 表层

### 水质：
- 水深 ≤10m → 表层
- 水深 >10m → 表层 + 底层

👉 用 mock 数据模拟水深
